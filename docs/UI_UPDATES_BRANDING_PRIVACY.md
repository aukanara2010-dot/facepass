# Обновления UI: Брендинг и Согласие на обработку данных

## Обзор изменений

Внесены важные изменения в пользовательский интерфейс для соответствия брендингу PIXORASOFT и добавления обязательного согласия на обработку персональных данных.

## Изменения брендинга

### 1. Замена названия компании
- **Было**: PHOTO POINT
- **Стало**: PIXORASOFT
- **Файлы**: `app/static/session/index.html`, `app/static/index.html`

### 2. Удаление кнопки "Личный кабинет"
- Убрана кнопка из шапки сайта
- Оставлен только переключатель языка (RU/EN)

### 3. Обновление описания
- **Было**: "Быстро, точно и совершенно бесплатно"
- **Стало**: "Быстро, точно и удобно"

### 4. Обновление копирайта
- **Было**: © 2024 PHOTO POINT
- **Стало**: © 2026 PIXORASOFT

## Добавление согласия на обработку данных

### 1. Чекбокс согласия
Добавлен обязательный чекбокс под кнопками действий:
```html
<input type="checkbox" id="privacy-agreement" class="...">
<label for="privacy-agreement">
    Я принимаю правила обработки персональных данных 
    и подтверждаю согласие с публичной офертой.
</label>
```

### 2. Блокировка кнопок
- Кнопки "Снять с камеры" и "Загрузить фото" изначально неактивны
- Добавлены классы: `opacity-50 cursor-not-allowed disabled`
- Кнопки активируются только после установки чекбокса

### 3. JavaScript функциональность
Добавлены функции:
- `toggleActionButtons()` - управляет состоянием кнопок
- Проверка согласия в `openCamera()` и `openFileDialog()`
- Показ предупреждения если согласие не дано

## Технические детали

### Состояния кнопок:
```javascript
// Неактивное состояние (по умолчанию)
disabled: true
classes: 'opacity-50 cursor-not-allowed'

// Активное состояние (после согласия)
disabled: false
classes: 'hover:shadow-lg hover:-translate-y-1'
```

### Проверка согласия:
```javascript
if (!this.privacyAgreement || !this.privacyAgreement.checked) {
    this.showToast('Пожалуйста, примите условия обработки персональных данных', 'warning');
    return;
}
```

## Пользовательский опыт

### До изменений:
1. Пользователь сразу мог нажать кнопки действий
2. Не было согласия на обработку данных

### После изменений:
1. Пользователь видит неактивные кнопки
2. Должен прочитать и принять условия
3. Только после этого может использовать функции поиска
4. При попытке использования без согласия - показывается предупреждение

## Тестирование

Для проверки функциональности:
1. Откройте `/session/{session_id}`
2. Убедитесь что кнопки неактивны
3. Попробуйте нажать - должно показаться предупреждение
4. Поставьте чекбокс - кнопки должны активироваться
5. Проверьте что поиск работает после согласия
